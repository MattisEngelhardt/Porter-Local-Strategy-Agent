# Porter Artifact Framework v2.0

This framework is mandatory for every PDF brief and PPTX deck Porter creates. It is not a
presentation "skill" and not optional decoration. It is Porter's internal artifact operating
system: synthesis, document preparation, shaping, and rendering all apply it before a file is
written.

## Purpose

Porter creates decision artifacts, not generic reports. Every artifact must be vivid, concrete,
attention-first, and source-grounded. The artifact should make the strategic situation easier to
see without adding facts that were not researched or provided in documents.

## Non-Negotiables

- No invented facts, dates, numbers, logos, claims, customer names, competitors, or conclusions.
- If evidence is missing, show the gap plainly. Do not hide uncertainty behind design.
- Start with the decision-relevant tension: what changed, why it matters, what the user should do.
- Use "so what" headlines. Topic labels are not allowed unless the slide/page is an appendix.
- Convert dense prose into visible structure: evidence anchors, decision callouts, risk/option
  frames, comparison tables, and source notes.
- Prefer exact numbers and named evidence already present in the analysis. Never "round up" or
  make a number look more precise.
- Every design element must carry meaning. Decorative filler is removed.

## PDF Brief Frame

1. Header: title, artifact type, language, date, and Porter framework marker.
2. Bottom line: first visible content block; management should understand the answer immediately.
3. Evidence anchors: compact chips extracted only from the analysis or cited sources.
4. Body: short sections ordered by importance, each with a claim-style heading.
5. Source list: visible enough that a reader can audit the claims.
6. Gaps: if source coverage is thin, state it directly in the artifact.

## PPTX Deck Frame

1. Title slide: concrete artifact title and purpose, not a generic theme.
2. Executive summary: the bottom line in 1-3 bullets.
3. Middle slides: one claim per slide, with short evidence-backed bullets or a table.
4. Decision slide: recommendation / option / watch item that can stand alone.
5. Appendix: sources and evidence gaps.
6. Visual shell: consistent rail, type marker, footer, logo when available, and restrained color.

## Attention Rules

- Use contrast to guide the eye: dark header/title moments, cyan action accents, neutral surfaces,
  and risk/positive colors only when they encode meaning.
- Show concrete anchors first: numbers, dates, named companies, source documents, or validated
  claims. If none exist, show the evidence gap instead of adding visual noise.
- Keep words tight. If a message needs more words, split it into another section or slide.

## Source Discipline

- A visual element may only use text already present in the analysis, source list, or supplied
  document names.
- A source chip can summarize source availability, but it cannot create a claim.
- Any material single-source or unverified claim must remain visibly qualified.
